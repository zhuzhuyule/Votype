# 智能总结与用户画像系统设计

> 设计日期：2026-01-24

## 概述

为 Votype 添加智能总结功能，帮助用户回顾每日/周/月的语音输入活动，并通过 AI 分析生成用户画像，反哺到 ASR 识别和 LLM 润色流程中。

## 需求背景

### 用户目标

1. **工作日志回顾** — 能回顾某天/周做了什么，类似工作日志
2. **效率追踪** — 了解自己用语音输入了多少内容，在哪些应用中使用最多
3. **自我反思** — 通过 AI 分析了解自己的沟通方式和表达习惯

### 核心特性

- 日/周/月三层粒度 + 自定义日期范围
- 轻量统计自动生成，深度 LLM 分析手动触发
- 本地 SQLite 缓存 + 可选导出 Markdown/JSON
- 用户画像数据反哺到润色流程，保持个人表达风格
- 可配置的反馈风格（客观中立/鼓励型/直接犀利）

### 画像数据应用场景

| 画像维度        | 应用到 ASR               | 应用到 LLM 润色          |
| --------------- | ------------------------ | ------------------------ |
| 常用专业术语    | 添加到热词表，提高识别率 | 保留专业词汇，不过度简化 |
| 表达风格        | —                        | 润色时保持一致风格       |
| 句式偏好        | —                        | 不改变用户习惯的句子结构 |
| 常用短语/口头禅 | 识别为固定搭配           | 保留个人特色表达         |

## 数据模型

### 新增数据库表

```sql
-- 1. 总结缓存表
CREATE TABLE summaries (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    period_type TEXT NOT NULL,        -- 'day' | 'week' | 'month' | 'custom'
    period_start INTEGER NOT NULL,     -- 时间戳
    period_end INTEGER NOT NULL,

    -- 自动统计（JSON）
    stats JSON NOT NULL,

    -- LLM 生成内容（可为空，手动触发后填充）
    ai_summary TEXT,
    ai_reflection TEXT,
    ai_generated_at INTEGER,
    ai_model_used TEXT,

    created_at INTEGER NOT NULL,
    updated_at INTEGER NOT NULL,

    UNIQUE(period_type, period_start, period_end)
);

-- 2. 用户画像表
CREATE TABLE user_profile (
    id INTEGER PRIMARY KEY CHECK (id = 1),  -- 单行，全局画像

    -- 统计型画像（自动更新）
    vocabulary_stats JSON,      -- 词频、专业术语等
    expression_stats JSON,      -- 句长、标点使用等
    app_usage_stats JSON,       -- 各应用使用分布
    time_pattern_stats JSON,    -- 使用时间规律

    -- LLM 分析型画像（定期更新）
    communication_style TEXT,   -- "直接简洁，技术导向..."
    tone_preference TEXT,       -- "中性偏正式"
    style_prompt TEXT,          -- 可直接注入 Skill 的提示词

    -- 用户偏好设置
    feedback_style TEXT DEFAULT 'encouraging',  -- 'neutral'|'encouraging'|'direct'

    last_analyzed_at INTEGER,
    updated_at INTEGER NOT NULL
);
```

## LLM Prompt 设计

### 1. 内容回顾生成 Prompt

```markdown
你是一个智能助手，帮助用户回顾和总结他们的语音输入记录。

## 输入数据

时间范围：{{period_start}} 至 {{period_end}}
记录数量：{{entry_count}} 条

按应用分组的转录内容：
{{#each by_app}}

### {{app_name}} ({{count}}条)

{{#each entries}}

- [{{time}}] {{text}}
  {{/each}}
  {{/each}}

## 任务

1. 为每个应用生成 2-3 条简短摘要，概括用户做了什么
2. 使用用户的原始表达风格，不要过度正式化
3. 突出关键活动和主题
4. 总字数控制在 200 字以内

## 输出格式

按应用分段，每段 2-3 个要点，使用 • 符号列表
```

### 2. 深度分析 Prompt

```markdown
你是一位沟通教练，帮助用户理解和改进他们的表达方式。

## 用户偏好的反馈风格

{{feedback_style_description}}

## 本期数据统计

- 总记录：{{entry_count}} 条
- 总字符：{{total_chars}} 字
- 平均句长：{{avg_sentence_length}} 字
- 使用应用：{{app_list}}

## 转录内容样本

{{content_samples}}

## 上期画像（如有）

{{previous_profile}}

## 任务

分析用户的沟通风格，输出以下内容：

### 1. 沟通风格 (80字以内)

描述用户的整体表达特征：正式/口语、简洁/详细、直接/委婉等

### 2. 表达特点 (3-5条)

具体的语言习惯，如常用词汇、句式偏好、语气特征

### 3. 反思与建议 (2-3条)

{{#if feedback_style == 'encouraging'}}
以积极鼓励的语气，先肯定优点，再温和建议
{{else if feedback_style == 'direct'}}
直接指出可改进的地方，不绕弯子
{{else}}
客观中立地描述，不带评价色彩
{{/if}}

### 4. 与上期对比 (如有上期数据)

指出明显的变化和趋势

### 5. 风格提示词 (供系统使用)

生成一段 50 字以内的风格描述，用于注入到润色 Prompt 中
格式：「用户偏好：...」
```

### 3. 反馈风格描述映射

| feedback_style | 描述注入                                                     |
| -------------- | ------------------------------------------------------------ |
| `neutral`      | 请保持客观中立，仅描述事实，不带褒贬评价                     |
| `encouraging`  | 请用积极鼓励的语气，先肯定用户的优点和进步，再温和地提出建议 |
| `direct`       | 请直接了当，重点指出可改进的地方，不需要过多铺垫             |

### 画像注入到润色流程

```rust
// 在现有的 post_process 逻辑中
async fn build_prompt_with_profile(
    base_prompt: &str,
    user_profile: Option<&UserProfile>,
) -> String {
    let mut prompt = base_prompt.to_string();

    if let Some(profile) = user_profile {
        if let Some(style_prompt) = &profile.style_prompt {
            // 在 system prompt 末尾追加用户风格提示
            prompt = format!("{}\n\n{}", prompt, style_prompt);
        }
    }

    prompt
}
```

## 风格提示词示例

当用户画像分析完成后，生成的 `style_prompt` 会被注入到润色 Skill 中：

```
用户偏好：
- 表达风格偏技术性，常用缩写如 API、PR、CR
- 喜欢用短句，平均每句 15 字
- 较少使用语气词，风格直接
- 常用连接词："然后"、"所以"、"其实"

请在润色时保持这些个人特征，不要过度正式化。
```
