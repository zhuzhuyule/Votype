# Auto-Generate Skill Design

## Overview

Add a "Voice Create" template option that allows users to create a complete skill automatically. Users provide raw content (via text or voice input), and the system automatically runs a 3-step LLM pipeline to generate polished instructions, name, icon, and description — saving the result without manual intervention.

## User Flow

1. User selects "Voice Create" from the template list (alongside Polish / Translate / Summary / AI Chat / Blank)
2. A dedicated dialog opens with a TextArea for input
3. User enters raw content via typing or voice input (focus is inside dialog, so existing transcription shortcuts work)
4. User clicks "Start Generate"
5. Dialog shows step-by-step progress as 3 LLM calls execute serially
6. On completion, the skill is auto-saved and the user is navigated to the edit page

## Pipeline Steps

| Step | Operation            | Backend Command                         | Input                          | Output                       |
| ---- | -------------------- | --------------------------------------- | ------------------------------ | ---------------------------- |
| 1    | Polish instructions  | `optimize_text_with_llm` (existing)     | Raw user input                 | Professional prompt template |
| 2    | Generate name + icon | `generate_skill_metadata` (new)         | Polished instructions + locale | `{ name, icon }`             |
| 3    | Generate description | `generate_skill_description` (existing) | Name + instructions + locale   | Description string           |

## New Backend Command

### `generate_skill_metadata`

- **Inputs**: `instructions: String`, `locale: Option<String>`
- **Output**: `Result<SkillMetadata, String>` where `SkillMetadata = { name: String, icon: String }`
- **Prompt template**: `system_skill_metadata.md` (new resource file)
- **Behavior**: LLM extracts a concise skill name and recommends a Tabler icon name from the instructions content
- **Output format**: JSON `{ "name": "...", "icon": "Icon..." }` parsed by backend

## UI Components

### AutoGenerateDialog

A Radix Dialog component containing:

- **TextArea**: For manual text input or voice-transcribed content
- **Start Generate button**: Triggers the pipeline
- **Cancel button**: Closes dialog, discards state
- **Progress stepper**: Shows 3 steps with states (pending / in_progress / completed / error)

```
Step display example:
  ✓ Polishing instructions       [Done]
  ● Generating name and icon     [In progress...]
  ○ Generating description       [Waiting]
```

### Template Entry

Add "Voice Create" option to the existing template list in `PromptsConfiguration.tsx` or the template selection component, with `IconMicrophone` icon.

## Completion Flow

After all 3 steps succeed:

1. Create a blank skill file via `create_skill_from_template("blank")`
2. Populate with generated name, instructions, description, icon
3. Save via `save_external_skill`
4. Close dialog
5. Navigate to the skill's edit page
6. User can modify and re-save manually

## Error Handling

- If any step fails: show error message, offer "Retry" button (retries from failed step)
- User can cancel at any time; unsaved results are discarded
- Network/LLM timeout: standard error toast
