## 数据处理指令

### 层次 A：标签累积（精确提取）

#### 1. 技能领域标签

从语音内容中提取技术关键词，并统计频次：

**编程语言**：

- Python, JavaScript, TypeScript, Java, Go, Rust, etc.

**框架/库**：

- React, Vue, Angular, Next.js, Express, Django, etc.

**工具/平台**：

- Docker, Kubernetes, Git, VS Code, Figma, etc.

**技术概念**：

- OAuth, JWT, RESTful API, GraphQL, Microservices, etc.

**输出格式**：

- "React +3, TypeScript +2, Docker +1"
- 数字表示今天出现的次数

#### 2. 主题标签

识别项目名称、产品名称、讨论话题：

**项目/产品**：

- 从语音中识别用户提到的项目名、产品名
- 例："项目Phoenix"、"用户中心模块"

**技术主题**：

- 技术选型、性能优化、UI设计、安全加固、etc.

**输出格式**：

- "用户认证模块 +5, API设计 +3, 性能优化 +2"

#### 3. 角色/场景标签

从应用使用和工作内容推断：

**工作角色**：

- 前端开发、后端开发、全栈开发、架构师、DevOps、产品经理、设计师

**常见场景**：

- 代码开发、代码评审、需求讨论、技术调研、文档编写、测试调试

**推断依据**：

- VS Code + React → 前端开发
- Terminal + Docker → 后端/DevOps
- Figma + Sketch → UI设计
- Chrome + 文档查阅 → 技术调研

**输出格式**：

- "角色推断：前端开发（基于 React + TypeScript + VS Code）"
- "常见场景：技术调研 +45%，代码开发 +35%，需求讨论 +20%"

### 层次 B：深度画像分析（风格提炼）

#### 4. 工作风格分析

基于今天的行为模式，分析工作风格：

**专注 vs 多任务**：

- 专注型：长时间单一任务，切换少
- 多任务型：频繁切换，并行处理

**独立 vs 协作**：

- 独立型：大部分时间在 IDE/编辑器，少沟通
- 协作型：频繁使用沟通工具（Slack, 会议）

**问题解决方式**：

- 查文档型：遇到问题先查官方文档、Stack Overflow
- 试错型：直接尝试，通过调试找答案
- 求助型：倾向于问同事/社区

**输出格式**：

- "工作风格：今天展现'探索尝试型'特征，遇到 OAuth 问题时采用'查文档+试错'并行策略"
- 必须有具体行为支撑，不能空洞判断

#### 5. 表达习惯分析

分析语音内容的表达特点：

**语言风格**：

- 技术细节型：深入讨论实现细节、代码逻辑
- 概念抽象型：更关注架构、设计模式、抽象思维
- 任务导向型：聚焦"做什么"、"完成什么"

**关注重点**：

- 实现细节（如：具体API参数、代码片段）
- 系统架构（如：模块划分、数据流）
- 用户体验（如：交互流程、视觉效果）
- 团队协作（如：分工、沟通、进度）

**输出格式**：

- "表达习惯：技术细节导向，今天多次深入讨论 token 存储、过期处理等实现细节，建议后续分析侧重技术深度"

#### 6. 学习与成长信号

识别新领域探索和技能深化：

**新领域探索**：

- 首次出现的技术词汇（与历史对比）
- 大量查阅新技术的文档
- 例："首次出现 OAuth 相关内容，标记为'新领域探索'"

**技能深化**：

- 从"使用"到"优化"到"原理"的演进
- 例："上周开始使用 React Hooks，今天深入研究其实现原理"

**输出格式**：

- "学习信号：首次出现 OAuth 2.0 深度内容，标记为新领域探索"
- "技能深化：React 从基础使用进入性能优化阶段"

## 质量标准

**优秀的画像更新**应该：

- ✅ 标签累积精确，包含频次
- ✅ 工作风格有具体行为支撑
- ✅ 表达习惯分析能指导后续分析策略
- ✅ 发现新领域探索和技能深化

- ❌ 避免空洞标签（"技术能力强"）
