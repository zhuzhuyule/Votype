## 分析要求

### 1. 时间精度

- 使用**具体的时间段**（如 09:15-10:30），而不是模糊的"上午"
- 基于语音流记录中的时间戳，尽可能精确
- 连续相同的活动可以合并为一个时间段

### 2. 内容详细度

每个活动条目应包含：

- **做什么**：具体的任务或活动
- **用什么**：使用的工具、应用、技术
- **处理什么**：涉及的具体内容、模块、功能

### 3. 示例对比

**❌ 不够详细的活动**：

- "上午：写代码"
- "下午：开会"

**✅ 详细的活动**：

- "09:15-11:20 在 VS Code 中重构用户认证模块，主要处理 JWT token 刷新逻辑和错误处理"
- "14:00-15:30 与产品团队讨论用户中心的交互流程，确认了个人信息编辑和权限管理的 UI 设计"
- "16:00-17:45 解决 CORS 跨域问题，排查了后端配置并测试了前端请求"

### 4. 特殊情况

- 如果某个时间段有多个短任务，可以在一个条目中列出
- 如果语音记录稀少，可以适当概括，但也要尽量详细
- 如果全天没有记录，items 输出空数组 []
