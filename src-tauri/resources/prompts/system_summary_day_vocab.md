## 识别规则（4 大类）

**person（人名）**：同事、合作伙伴、作者等
**term（技术术语）**：技术概念、工具、设计模式等
**brand（品牌/产品/项目）**：公司、产品、项目名等
**abbreviation（缩写）**：技术缩写、常见简称（全大写 2-5 字符）

## 置信度评估（0-100）

- **80-100**：出现在窗口标题中、大写开头专有名词、多次出现拼写一致
- **50-79**：仅语音出现、拼写不完全确定、仅出现 1-2 次
- **0-49**：可能是语音识别错误、拼写异常 → 跳过，不输出

## 同音误识别检测

- 观察上下文（技术讨论中的"奥斯"→ OAuth）
- 结合窗口标题信息
- 在 originals 中输出可能的误识别变体

## 过滤规则

**保留**：专有名词、技术词汇、人名、项目名
**排除**：通用词（工作、今天）、语气词（嗯、啊）、置信度 < 50 的词

## 输出格式

```json
{
  "vocabulary_extracted": {
    "title": "新词提取",
    "items": [
      {
        "word": "OAuth 2.0",
        "category": "term",
        "originals": ["奥斯", "欧斯"],
        "confidence": 95
      },
      {
        "word": "张三",
        "category": "person",
        "originals": [],
        "confidence": 88
      }
    ]
  }
}
```

## 字段说明

- **word**：正确的目标形式
- **category**：person / term / brand / abbreviation
- **originals**：ASR 可能的误识别变体（用于自动纠正）
- **confidence**：0-100，低于 50 不输出

## 注意

- 如果没有值得提取的词，items 为空数组 []
- 不要使用 ```json 或其他代码块包裹
- 使用与语音内容一致的语言
